| Category | Grade |
|----------|-------|
| **Architecture** | **C+** |

## Architecture Summary

### Strengths
- Modular separation: routers → services → models
- Async-first FastAPI design
- Docker containerization with multi-service composition
- Security fundamentals present (encryption, JWT, bcrypt)

### Issues by Priority

#### Critical
- **Dependency versions not pinned** (D+) — requirements.txt has almost no version constraints. SQLAlchemy, FastAPI, etc. could break on any `pip install`.

#### High
- **No database migration framework** — 11 ad-hoc migration functions in `start.py` using raw SQL. Should use Alembic.
- **State management fragile** — 5+ separate dicts tracking FFmpeg/timeline state with no locking (`asyncio.Lock`) and no database persistence.
- **Fire-and-forget async tasks** — Background tasks created but not monitored for failures.
- **No graceful shutdown handling** — `_shutdown_event` created but never triggered on SIGTERM.

#### Medium
- **Low test coverage** — 4 test files (22 tests) for 8000+ lines of service code. Missing integration tests.
- **Duplicated logic** — Password decryption, FFmpeg command building repeated across files.
- **Service layer bloat** — `ffmpeg_manager.py` (852 lines), `timeline_executor.py` mix multiple concerns.
- **No structured logging** — Emoji in log messages, no JSON format for aggregation.
- **Missing API versioning** — No `/api/v1/` prefix.
- **Inconsistent error responses** — Raw exception strings in HTTP 500 responses.

## Security Issues

### CRITICAL (Fix immediately)

1. **Preview endpoints unprotected** — `routers/preview.py` has NO auth dependency. Anyone can start/stop timelines and streams via `/api/preview/*`.

2. **CORS allows CSRF** — `allow_credentials=True` + `allow_methods=["*"]` + `allow_headers=["*"]` in `main.py:155-162`. Combined with regex origin that matches any IP and `*.vistter.com`.

3. **Stream keys stored in plaintext** — `destination.stream_key` is a plain `Column(String)` with no encryption. Also returned in API responses.

4. **YouTube API key stored in plaintext** — `destination.youtube_api_key` is unencrypted.

5. **Default admin reset script** — `migrations/reset_admin_password.py` resets to admin/admin.

6. **.env with live secrets in git** — Cloudflare tunnel token, JWT_SECRET_KEY, ENCRYPTION_KEY may be in git history.

7. **Database not encrypted at rest** — SQLite file on Docker volume, no LUKS or encryption.

### HIGH (Fix within 1 week)

8. **No rate limiting on login** — `/api/auth/login` can be brute-forced. No SlowAPI or similar.

9. **OAuth state parameter is predictable** — Uses sequential destination ID as `state`, no CSRF token.

10. **PKCE not enforced** — `code_verifier` is optional and may not be generated by google-auth-oauthlib.

11. **RTMP relay allows all** — `nginx.conf`: `allow publish all; allow play all;` — anyone on the network can publish.

12. **API docs publicly accessible** — `/api/docs` and `/api/redoc` exposed via Cloudflare tunnel.

13. **Docker runs as root** — No `USER` directive in backend Dockerfile.

### MEDIUM (Fix within 1 month)

14. **No audit logging** — No trail for who changed what (deleted destinations, changed passwords).

15. **No request timeout** — uvicorn has no timeout; slow client attacks possible.

16. **No secrets rotation mechanism** — JWT_SECRET_KEY, ENCRYPTION_KEY have no versioning.

17. **Snapshot URLs not validated** — Could be SSRF attack vector.

18. **Build tools in production image** — `build-essential` installed but not needed at runtime.

## Action Plan

### Phase 1: Critical Security (Today)
- [ ] Add auth to preview endpoints
- [ ] Fix CORS: whitelist methods [GET,POST,PUT,DELETE], whitelist headers [Content-Type, Authorization], restrict origins
- [ ] Encrypt stream_key in database (use existing Fernet encryption)
- [ ] Encrypt youtube_api_key in database
- [ ] Remove or protect admin reset script
- [ ] Rotate all secrets (JWT_SECRET_KEY, ENCRYPTION_KEY, Cloudflare token)

### Phase 2: High Priority Security (This week)
- [ ] Add rate limiting (SlowAPI) on /api/auth/login
- [ ] Add CSRF token to OAuth state parameter
- [ ] Restrict RTMP relay to backend only
- [ ] Disable API docs in production
- [ ] Add USER directive to Dockerfile
- [ ] Don't return stream_key in API responses
